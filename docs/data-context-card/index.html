声明式 data-context-card 协议

零依赖、框架无关。
元素声明触发、模板定义内容、智能定位投影。

为AI自动化生成而设计LLM 只需声明 data-context-card + <template> ，无需操心交互与定位
Template Projection Smart Positioning Auto Dark Mode Dynamic DOM Framework Agnostic

🚀 三步集成

1

引入框架

通过 <script defer> 加载，内联执行初始化。

<script defer src="data-context-card.umd.js" onload="DataContextCard.init()"></script>
2

定义模板

创建 <template> 作为投影内容。

<template id="card"> <div>投影内容</div> </template>
3

声明触发

给触发元素添加 data-context-card 属性，值为模板 ID。

<span data-context-card="card">悬停查看</span>

以下四个示例基于真实的 data-context-card 协议。将鼠标悬停在触发区域上即可看到上下文投影卡片。

🎯 核心协议 （必须了解）

属性说明
data-context-card="id"标记触发元素，值为 <template> 的 ID
data-context-card-size="N"覆盖默认 25vw 尺寸（如 40 → 40vw）
<template id="id">定义投影内容，框架自动克隆到浮动卡片
DataContextCard.init()激活运行时，扫描所有 [data-context-card] 并绑定

⚙️ 常用配置 （建议了解）

初始化前修改 DataContextCard.config：

配置项默认值说明
offset12卡片与触发元素的间距 (px)
edgePadding16卡片与视口边缘的最小间距 (px)
hideDelay120隐藏延迟 (ms)
DataContextCard.config.offset = 16 DataContextCard.init()

自定义事件

事件说明
dcc:mount卡片渲染完成时触发。外部框架（React/Vue 等）可监听此事件，获知何时将子组件挂载到卡片 DOM 内。
e.detail.mode === 'template' 表示内容来自模板克隆；'portal' 表示未找到模板，等待外部注入。
dcc:unmount卡片即将隐藏时触发，外部框架在此事件中执行清理（卸载 Portal 子组件、释放资源等）
document.getElementById('data-context-card-container') .addEventListener('dcc:mount', (e) => { console.log('mode:', e.detail.mode) // 'template' | 'portal' })

框架集成示例
HTML (UMD)

ESM
import DataContextCard from 'data-context-card' DataContextCard.init()

useEffect(() => { DataContextCard.init() return () => DataContextCard.destroy() }, [])
Vue
onMounted(() => DataContextCard.init()) onUnmounted(() => DataContextCard.destroy())
其他
data-portal — 标记框架不管理的 DOM 容器。当模板内部有 data-portal 元素时，卡片隐藏不会清空该容器内容，确保外部框架（React Portal、Vue Teleport 等）渲染进去的子组件在显示/隐藏切换中保持状态不丢失。
data-dcc-close — 模板内关闭按钮标记
📱 移动端自动识别触摸设备，点击切换显示/隐藏
📘 内置 TypeScript 类型声明，无需 @types
🌍 支持 Chrome · Edge · Safari · Firefox 等所有现代浏览器
